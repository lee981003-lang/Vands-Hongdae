# AGENTS.md — 협업 규칙

> 이 파일은 **Claude**와 **Codex**가 이 프로젝트에서 협업하는 방식을 정의한다.
> 작업 시작 전 반드시 이 파일을 읽고, 아래 규칙을 따른다.

---

## 1. 역할 분담 (가장 중요)

| 에이전트 | 역할 |
|---|---|
| **Claude** | **계획 담당.** 계획 수립, 추가·수정, 설계 결정 정리, 리스크 검토. 직접 구현 코드는 작성하지 않는다. |
| **Codex** | **구현 담당.** Claude가 수립·확정한 계획을 **기반으로** 실제 프로그램을 구현한다. 계획 자체를 새로 바꾸지 않는다. |

- 계획을 바꿔야 할 일이 생기면 → **Claude가 계획을 수정**하고 Notion에 반영한 뒤, Codex가 그 변경분을 읽고 구현한다.
- Codex는 구현 중 발견한 문제·제안을 **계획 변경 요청**으로 Claude에게 전달한다(직접 계획 문서를 갈아엎지 않는다).

---

## 2. 기록 허브 (Single Source of Truth)

- **모든 활성 계획·결정·구현 기록은 Notion 기록 허브에 남긴다.**
- 허브: **[🩺 실시간 시술 현황 대시보드](https://app.notion.com/p/385fe9425b5581c0b876c18339d8a673)**
  - Page ID: `385fe942-5b55-81c0-b876-c18339d8a673`
- 기존 계획·결정 로그·구현 완료 문서는 **원문 보존용 참고 자료**다. 삭제·덮어쓰기·소급 이관하지 않고, 새 허브에서는 링크로만 참조한다.
- 구현의 유일한 승인 기준은 허브의 **「⚖️ decisions」** 페이지에서 Claude가 `Confirmed`로 확정한 Decision이다.

### 하위 페이지와 작성 책임

| 페이지 | 작성자 | 용도 |
|---|---|---|
| `📌 현재 상황 정리 & 실행 계획` | Claude | 현재 상태, 다음 작업, 차단 사항, 원문 링크 |
| `📌 memory` | Claude | 장기 아키텍처·권한·배포 제약 |
| `📅 daily` | 해당 작업자 | 날짜별 짧은 요약과 관련 기록 링크 |
| `💡 insights` | Claude | 코드·문서·운영에서 발견한 사실과 위험. 해결안은 확정하지 않음 |
| `⚖️ decisions` | Claude | 구현 전 확정 설계와 완료 조건 |
| `📜 scripts` | Codex | 확정 Decision의 구현 결과와 검증 증적 |

- Insight ID는 `I-001`, Decision ID는 `D-001`부터 순차적으로 사용한다.
- Decision에는 최소한 해결할 Insight, 결정 내용, 코드 수정안, 지침서 수정안, 영향 범위, 완료 조건/검증 기준, 상태를 남긴다.
- Script에는 반드시 Decision ID, 변경 파일, 핵심 변경, 테스트 결과, 완료 여부, 남은 리스크를 남긴다.
- Codex는 `Confirmed` Decision 없이 구현하거나 Script에 완료 기록을 남기지 않는다.

## 3. 읽기 규칙 (필수)

> **전체를 읽지 않는다. 항상 필요한 부분만 좁게 읽고 진행한다.**

- 계획이나 수정사항을 읽을 때 **페이지 전체를 통째로 읽지 말 것.**
- 구현 전에는 해당 `D-번호` Decision과 연결된 Insight, 필요한 memory 항목만 읽는다.
- 구현 후에는 같은 `D-번호`의 Script와 당일 daily 기록만 추가·확인한다.
- 예: 데이터 모델 작업 → 해당 Decision + 관련 Insight + memory의 배포 제약만.
- 예: 최근 확정사항 확인 → `⚖️ decisions`의 해당 `D-번호`만.
- 이유: 컨텍스트 절약 + 작업 정확도 유지 + 불필요한 재해석 방지.
- Notion `fetch`/검색 시에도 관련 키워드·섹션 단위로 좁혀서 가져온다.

---

## 4. 작업 흐름

```
Claude(insight 기록) → Claude(Decision Confirmed) → Codex(구현·Script 기록)
        ↑                        │                           │
        └──── 계획 변경 요청 ────┴──── daily 링크·요약 ───────┘
```

1. Claude가 발견 사실과 위험을 `💡 insights`에 기록한다.
2. Claude가 해결안을 `⚖️ decisions`에 작성하고 상태를 `Confirmed`로 확정한다.
3. Codex는 해당 `D-번호`와 연결된 항목만 읽고 구현한다.
4. Codex는 검증 후 `📜 scripts`에 구현 결과를 기록하고, 당일 `📅 daily`에는 링크와 짧은 요약만 남긴다.
5. 구현 중 계획과 어긋나거나 새 결정이 필요하면 Codex는 구현을 멈추고 Claude에게 새 Insight/Decision 기록을 요청한다.
